3. Scenario APIs

Create:

GET /api/simulation/scenarios

Returns all available predefined scenarios.

Create:

POST /api/simulation/run/<scenario_id>

Runs one selected scenario.

Rules:
- scenario_id must exist.
- Generated readings must include valid sensor_id, sensor_type, value, unit, timestamp.
- The generated readings must be processed through the same existing reading pipeline used by:
  POST /api/sensors/<sensor_id>/reading
- Scenario readings must be validated.
- Scenario readings must be saved in readings.json if valid.
- Alerts must be generated if thresholds are breached.
- Critical alerts must trigger simulated automation.
- Scenario results must be returned in the API response.

Example response:

{
  "success": true,
  "scenario_id": "dry_soil",
  "scenario_name": "Dry Soil",
  "message": "Simulation scenario completed successfully",
  "generated_readings": [...],
  "alerts": [...],
  "automation_actions": [...],
  "expected_result": "Low soil moisture alert and irrigation response"
}
