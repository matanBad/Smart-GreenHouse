1. Threshold Management APIs

The following APIs should already exist from Prompt 3:

GET /api/thresholds
GET /api/thresholds/<sensor_type>

Keep them working.

Add:

PUT /api/thresholds/<sensor_type>

Request body example:

{
  "min_value": 18,
  "max_value": 30,
  "warning_margin": 2,
  "role": "system_administrator"
}

Rules:
- Only role = "system_administrator" is allowed to update thresholds in this stage.
- If role is missing or not system_administrator, reject the request safely.
- sensor_type must be one of:
  - temperature
  - humidity
  - soil_moisture
  - light_intensity
- min_value must be numeric.
- max_value must be numeric.
- warning_margin must be numeric and greater than or equal to 0.
- min_value must be lower than max_value.
- Save only the rule that belongs to the selected sensor_type.
- Do not affect other threshold rules.
