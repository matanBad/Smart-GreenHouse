6. Alert History API

Create:

- active
- resolved

Ordered by created_at newest to oldest.

7. Sensor-Specific Alerts API

Create:

GET /api/sensors/<sensor_id>/alerts

Returns all alerts related to a specific sensor.

If the sensor does not exist, return a safe 404 response.

8. Resolve Alert API

Create:

PATCH /api/alerts/<alert_id>/resolve

Behavior:
- Find the alert by alert_id.
- If the alert does not exist, return 404.
- If the alert is already resolved, return a safe response.
- If active, set:
  status = resolved
  resolved_at = current timestamp

Resolved alerts:
- must disappear from active alerts
- must remain in alert history
