2. Sensor Management APIs

Keep existing APIs working:

GET /api/sensors
GET /api/sensors/<id> if already exists

Add or update:

GET /api/sensors/status

Returns all sensors with:
- sensor_id
- sensor_name
- sensor_type
- unit
- last_valid_reading_at
- expected_frequency_seconds
- communication_status
- current reading if available

PUT /api/sensors/<sensor_id>

Allows demo administrator update of sensor configuration.

Request body example:

{
  "sensor_name": "Main Temperature Sensor",
  "unit": "C",
  "status": "enabled",
  "role": "system_administrator"
}

Rules:
- Only role = "system_administrator" can update sensor configuration in this stage.
- sensor_id cannot be changed.
- sensor_type cannot be changed.
- sensor_name can be updated.
- unit can be updated only if valid for that sensor type.
- status can be updated only to supported values:
  - enabled
  - disabled
- Invalid updates must be rejected safely.
- Successful configuration changes must be logged.

3. Sensor Service Update

Update sensor_service.py.

Implement clean functions:

- get_all_sensors()
- get_sensor_by_id(sensor_id)
- update_sensor_configuration(sensor_id, update_data, role)
- validate_sensor_update(sensor_id, update_data, role)
- update_sensor_last_valid_reading(sensor_id, timestamp)
- calculate_sensor_activity_status(sensor)
- get_all_sensor_statuses()
- mark_sensor_inactive_if_stale(sensor)

Activity logic:
- If a sensor has no last_valid_reading_at, communication_status = no_data.
- If current time - last_valid_reading_at <= expected_frequency_seconds * 2, communication_status = active.
- If current time - last_valid_reading_at > expected_frequency_seconds * 2, communication_status = inactive.

4. Update Reading Submission Flow

Update:

POST /api/sensors/<sensor_id>/reading

After a reading passes validation and is saved:
- update the sensor last_valid_reading_at
- recalculate communication_status
- save the updated sensor object in sensors.json

Invalid or rejected readings:
- must NOT update last_valid_reading_at
- must NOT set communication_status to active
