1. Threshold Rule Structure

Use data/threshold_rules.json as the source of threshold rules.

Each rule should include:
- rule_id
- sensor_type
- min_value
- max_value
- warning_margin
- unit
- severity_levels

Supported sensor types:
- temperature
- humidity
- soil_moisture
- light_intensity

Example structure:

{
  "rule_id": "RULE-TEMP-001",
  "sensor_type": "temperature",
  "min_value": 18,
  "max_value": 30,
  "warning_margin": 2,
  "unit": "C",
  "severity_levels": ["normal", "warning", "critical"]
}

2. Rule Engine Logic

Implement rule_engine.py with clean functions:

- get_threshold_rule(sensor_type)
- evaluate_reading_status(sensor_type, value)
- is_below_min(value, rule)
- is_above_max(value, rule)
- is_near_threshold(value, rule)
- apply_threshold_evaluation(reading)

Status logic:

A reading is "critical" if:
- value < min_value
or
- value > max_value

A reading is "warning" if:
- value is still inside the allowed range
but close to min_value or max_value according to warning_margin

A reading is "normal" if:
- value is inside the allowed range
and not close to the warning area

Examples:
- Temperature min=18, max=30, warning_margin=2
- 24 → normal
- 19 → warning
- 29 → warning
- 31 → critical
- 15 → critical
