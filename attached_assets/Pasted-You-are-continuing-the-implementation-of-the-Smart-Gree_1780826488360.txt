Main Requirements to Implement:

1. Sensor Reading Submission API

Create the endpoint:

POST /api/sensors/<sensor_id>/reading

The endpoint should accept JSON payloads with:

{
  "sensor_id": "TEMP-001",
  "sensor_type": "temperature",
  "value": 31.5,
  "unit": "C",
  "timestamp": "2026-05-11T10:30:00Z"
}

Rules:
- The sensor_id in the URL must match the sensor_id in the request body.
- The sensor must exist in sensors.json.
- The sensor_type must match the configured sensor type.
- The value must be numeric.
- The unit must match the configured unit.
- The timestamp must exist and be valid.
- Invalid readings must be rejected.
- Invalid readings must NOT be saved in readings.json.
- Invalid readings must be written to system_logs.json.

2. Validation Service

Implement validation_service.py with clear functions:

- validate_sensor_exists(sensor_id)
- validate_sensor_type(sensor_id, sensor_type)
- validate_numeric_value(value)
- validate_unit(sensor_id, unit)
- validate_timestamp(timestamp)
- validate_sensor_reading(sensor_id, payload)

The validation function should return a structured result:

{
  "is_valid": true/false,
  "errors": []
}

Security requirement:
Never trust client input.
Validate every field before saving.
